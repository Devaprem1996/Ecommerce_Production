# Ecommerce Production Project

Version: 1.0

Owner: <Devaprem>

Project Duration: 2 Weeks

Project Type: Production Ready

---

# Mission

Build a production-quality ecommerce application similar to Amazon or Flipkart using modern software engineering practices.

The project must be:

* Secure
* Scalable
* Maintainable
* Tested
* Deployable
* Well documented

---

# Development Principles

## Rule 1

Understand before building.

Never start writing code without understanding:

* What is required
* Why it is required
* How it fits into the application

---

## Rule 2

One feature at a time.

Example

❌ Login + Products + Cart together

✅ Login only

After Login

Test

Fix

Document

Commit

Then continue.

---

## Rule 3

Every feature follows the same lifecycle.

Requirement

↓

Planning

↓

Design

↓

Implementation

↓

Testing

↓

Documentation

↓

Git Commit

↓

Code Review

↓

Completed

---

## Rule 4

No copied code.

Every generated code block must be:

* Read
* Understood
* Explained
* Tested

before moving on.

---

## Rule 5

No production shortcuts.

Never disable:

* Authentication
* Validation
* Authorization
* Error handling
* Logging

just to make something work.

# Documentation Rules

Every major decision must be documented.

Every bug must be documented.

Every completed feature must be documented.

---

# Git Rules

Commit often.

Small commits.

Meaningful commit messages.

Example

feat(auth): implement Google OAuth

NOT

update

---

# AI Rules

AI must never:

* Skip steps
* Assume configuration exists
* Generate huge files without explanation
* Modify unrelated files

AI must always:

Explain

↓

Plan

↓

Generate

↓

Review

↓

Test

---
